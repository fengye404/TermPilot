# Why TermPilot

这份文档不讲安装步骤，只讲一件事：

**如果你本来就有很多方式能远程碰到一台机器，为什么还要用 TermPilot？**

答案不是“它也能远程进终端”，而是：

**它把“离开电脑以后，继续接上原来那条还在跑的终端会话”这件事做成了主路径。**

## 1. 最核心的优势：同一条会话不会分叉

很多远程方案都能让你“再进一条终端”。

但在真实工作里，你经常更需要的是：

- 电脑前已经有一条会话在跑
- 里面可能是 Claude Code、OpenCode、脚本、部署或批处理
- 你临时离开电脑
- 你不想重新开一条新的上下文
- 你想继续看原来那条会话，必要时补一条命令、发一个快捷键、或者把它停掉

TermPilot 的第一个优势，就是把这个需求变成默认能力：

- 电脑和手机看到的是同一条 `tmux` 会话
- 手机上补命令，写回的是原来的会话，不是另一条新 shell
- 回到电脑前，还是那条上下文，不需要重新找状态

对长期任务来说，这比“远程开个终端”重要得多。

## 2. 第二个优势：更适合持续运行的任务

TermPilot 的强项不是一次性远程操作，而是**持续运行中的任务连续性**。

它特别适合这些场景：

- AI 编码 agent 正在流式输出
- 一个部署流程要跑很久
- 一个脚本还没结束，但你已经离开当前桌面
- 你只想不时看一眼进度，再补一个轻量操作

这类场景的共同点是：

- 会话本身要一直活着
- 人不一定一直坐在电脑前
- 中途查看和轻控制比“重新进入系统”更重要

TermPilot 把这种工作流做得更顺手，因为它本来就是围绕“长期会话”设计的。

## 3. 第三个优势：手机端更像“任务面板”，不是缩小版桌面

很多方案在手机上能用，但体验并不是真的为手机设计的：

- 要么像缩小版桌面
- 要么像一堆系统管理面板
- 要么在手机上需要做太多重度输入

TermPilot 的取舍更明确：

- 先看会话列表
- 再点进某个会话看输出
- 有需要时发一条短命令
- 有需要时发快捷键
- 有需要时关掉会话

也就是说，它更像：

- 任务观察器
- 会话控制面板
- 长期终端任务的移动入口

而不是：

- 完整替代电脑桌面
- 在手机上做重度终端编辑

这种定位带来的优势是，移动端能更快进入状态，也更符合真实使用频率。

## 4. 第四个优势：使用成本低，部署路径短

TermPilot 刻意把用户路径收得很短：

- 服务器：`termpilot relay`
- 电脑：`termpilot agent`
- 手机：直接打开域名

第一次配对后，后续日常路径通常就是：

- 电脑执行 `termpilot agent`
- 电脑执行 `termpilot claude code`
- 手机上打开网页继续看

它的价值不只是“命令少”，而是：

- 不要求手机安装 App
- 不要求用户理解复杂的三端工程结构
- 不要求每次都重新组织远程访问路径

对个人项目来说，这种低操作成本本身就是优势。

## 5. 第五个优势：电脑和手机都不用暴露端口

TermPilot 的连接模型是：

- 电脑主动连 relay
- 手机主动连 relay
- relay 做中继、配对和权限边界

这带来的直接好处是：

- 家用电脑更容易接入
- 不需要让电脑自己暴露入站端口
- 不需要让手机进入任何复杂网络环境
- 云端顺手就能承载网页、授权、最近输出缓冲和会话元数据

这让它很适合“主力电脑在一个地方，人经常在外面”的使用方式。

## 6. 第六个优势：它把真正高频的事情做成了默认路径

TermPilot 不是想覆盖所有远程场景，它只想把最常见、最有价值的那条路径做顺。

对这个项目来说，那条路径是：

1. 电脑上启动 agent
2. 跑一个长期任务
3. 手机上继续看同一条会话
4. 轻量补命令或中断
5. 回到电脑前继续接着做

这个路径如果做顺了，TermPilot 就有价值。

这也解释了为什么它会有这些取舍：

- 用 `tmux` 作为会话底座
- 强调“只有体系内会话才同步”
- 手机端优先做会话列表和终端查看
- relay 同时负责授权和 Web UI 托管

这些不是技术炫耀，而是为了让主路径更短、更稳定。

## 7. 哪些人会最明显感到它的价值

### AI 编码用户

这是最直接的目标用户：

- Claude Code 在电脑前跑着
- 你需要离开工位
- 手机上继续看输出和状态
- 必要时补一条命令或直接停掉

这类用户最容易感受到“共享同一条会话”的价值。

### 长时间脚本和部署任务用户

如果你经常有这些任务：

- 构建
- 部署
- 迁移
- 抓取
- 批处理

你通常不需要在手机上做复杂编辑，但你很需要：

- 看它还在不在跑
- 看最新输出
- 做一次轻量控制

TermPilot 就是为这种使用密度设计的。

### 个人开发者和小团队

它尤其适合：

- 想要自己的终端工作流可跨端继续
- 不想引入重型远程桌面
- 不想先搭一堆复杂基础设施
- 更在意“顺手”和“持续性”而不是“大而全”

## 8. 它不适合解决什么

TermPilot 也不是万能工具。

下面这些目标，不是它的强项：

- 远程桌面
- 图形界面控制
- 在手机上做长时间重度终端输入
- 自动接管任意历史终端标签页
- 通用的多租户服务器运维平台

如果你的需求主要在这些方向，TermPilot 不一定是最优解。

它的优势来自聚焦，而不是包打天下。

## 9. 一句话总结

TermPilot 的优势，不在于“也能远程进终端”。

它真正的优势是：

**让一条正在运行的长期终端会话，可以在电脑和手机之间持续共享，而且这件事足够轻、足够顺手。**
