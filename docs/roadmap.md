# 持续改进计划

这份文档不是“推翻现有产品重做一遍”的路线图。

TermPilot 当前的主路径已经明确：

- relay 提供网页、WebSocket、配对与授权
- agent 管理本地 tmux 会话
- 手机端围绕查看、轻输入和快捷控制展开

后续改进的目标，是把这条主路径做得更稳、更顺、更容易长期使用。

## 1. 改进原则

后续迭代优先遵守这几条：

- 不改变产品核心定位：共享同一条长期终端会话
- 优先优化可靠性、易用性和一致性
- 不为了“看起来先进”破坏当前已经跑通的路径
- 先把单用户、个人设备场景做扎实

## 2. 近期重点

### 2.1 会话稳定性

继续优化当前 tmux 快照同步链路：

- 颜色和样式一致性
- 不同终端宽度下的布局稳定性
- replay 后的画面连续性
- 长时间运行下的状态漂移控制

### 2.2 移动端体验

继续围绕“查看 + 轻控制”优化，而不是把手机做成桌面 IDE：

- 更清晰的会话层级
- 更稳的快捷操作布局
- 专注模式与窄屏工作区
- 更好的输入和粘贴体验

### 2.3 部署与运维

让 relay 和 agent 更适合长期运行：

- 更清楚的健康检查与日志路径
- 更稳的升级流程
- 更明确的存储模式说明
- 更容易接入反向代理和进程管理器

### 2.4 测试覆盖

继续补强真实使用路径的验证：

- UI smoke
- 长时间同步稳定性
- 多设备隔离
- 版本升级后的兼容性检查

## 3. 中期改进

在不改变产品核心形态的前提下，继续把现有架构做细：

### 3.1 同步效率

- 降低 `tmux capture-pane` 轮询成本
- 减少不必要的重复帧传输
- 优化活跃会话和非活跃会话的同步策略

### 3.2 会话模型

- 把会话类型和退出语义表达得更明确
- 让 `shell` 会话与托管命令会话的体验更一致
- 提升 Web UI 对当前会话状态的解释能力

### 3.3 运维能力

- 更完整的状态诊断信息
- 更清楚的 grants / audit 管理路径
- 更成熟的默认部署建议

## 4. 安全强化

安全方向会继续推进，但在文档和产品定位里，它应该是“逐步强化当前模型”，而不是压过现有产品本身。

近期更现实的方向包括：

- 更清楚的 token 管理和默认值说明
- 更明确的公网部署安全建议
- 更可操作的 grants 撤销和设备管理流程
- 更稳定的本地状态目录与恢复策略

## 5. 远期方向

远期仍然有两条值得保留的方向，但它们不应该喧宾夺主：

- 更本地优先的数据组织
- 更强的端到端加密能力

这两条方向只有在不破坏当前主路径的前提下，才值得继续推进。

## 6. 一句话总结

TermPilot 接下来的重点不是“换一个产品方向”，而是：

**把现在这条已经成立的共享终端会话工作流，继续打磨成一个更稳定、更成熟、更适合长期使用的产品。**
