# TermPilot 用户流程

## 1. 第一版的基本使用方式

第一版不追求接管任意已有终端，而是要求用户从 TermPilot 管理的入口启动任务。

推荐工作流：

1. 在电脑上启动 TermPilot agent
2. 在电脑上用 TermPilot 命令创建一个会话
3. 用本地终端附着到这个会话
4. 在这个会话里运行 Claude Code、OpenCode 或其他任务
5. 出门后用手机打开同一个会话，继续查看输出和发送命令

## 2. 电脑端流程

### 2.1 启动 agent

agent 是一个常驻进程，负责：

- 与中继服务保持连接
- 发现和同步 TermPilot 管理的会话
- 把会话输出同步到手机
- 接收手机发来的控制消息

### 2.2 在电脑上创建会话

电脑端创建会话的作用不只是本地使用，还意味着：

- 这个会话会进入统一会话池
- 手机端会看到这个会话
- 手机端可以关闭它、继续输入它

### 2.3 在电脑上附着会话

本地附着的本质是连接到底层 `tmux` 会话。

这能保证：

- 电脑和手机看到的是同一个真实会话
- 两边输出是同步的
- 用户回到电脑前时可以继续无缝接手

## 3. 手机端流程

### 3.1 连接到中继服务

手机端打开网页后：

- 连接中继服务
- 获取在线设备信息
- 拉取统一会话池

### 3.2 查看会话列表

手机端应能看到：

- 当前有哪些会话
- 哪些会话仍在运行
- 当前会话的大致状态

### 3.3 打开某个会话

打开会话后，手机端会：

- 请求最近输出补拉
- 开始接收后续输出更新
- 持续显示会话的最新内容

### 3.4 在手机上发送控制

手机端至少应支持：

- 发送普通命令
- 发送 `Enter`
- 发送 `Ctrl+C`
- 关闭会话

## 4. 跨端同步流程

### 4.1 电脑创建，手机查看

流程如下：

1. 电脑端创建会话
2. agent 把新会话同步到中继服务
3. 手机端收到会话状态变化
4. 手机打开该会话并看到同样输出

### 4.2 手机创建，电脑接手

流程如下：

1. 手机端请求创建会话
2. 中继服务把请求转发给 agent
3. agent 在电脑上创建底层 `tmux` 会话
4. 电脑端可以通过本地命令附着进去

### 4.3 任意一端关闭会话

流程如下：

1. 手机或电脑发起关闭
2. 会话被标记退出并关闭底层 `tmux`
3. 另一端收到状态变化

## 5. 第一版明确不支持的流程

这些流程不在第一版范围内：

- 打开 agent 之前已经存在的历史终端自动接管
- 用户手动在普通 Terminal 或 iTerm 新开的体系外终端自动接管
- 手机端像桌面一样编辑文件
- 手机端接管完整图形界面

## 6. 为什么要坚持“统一会话池”

因为这是整个产品的核心：

- 统一会话池能保证手机和电脑看到同一个对象
- 会话的创建来源不再重要，重要的是它是否被纳入系统
- 这比“远程再开一个新的终端”更符合你的真实工作流

## 7. 第一版成功的判断标准

如果下面这些动作都成立，就说明第一版方向是对的：

- 在电脑上创建并运行一个智能体任务
- 手机能稳定看到同一份持续输出
- 手机能中断或追加命令
- 用户回到电脑前后能继续接着同一个会话工作
