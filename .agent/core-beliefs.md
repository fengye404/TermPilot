# Core Beliefs

- Favor clear runtime ownership over convenience imports.
- Keep the product narrow: TermPilot is about continuity of one managed session, not remote desktop control.
- Prefer explicit boundaries and checks over tribal knowledge.
- Treat the repository as the durable system of record for engineering context.
- Keep user-facing docs and internal agent docs separate, but consistent.
- Default to lightweight, repeatable verification loops that agents can run without extra narration.
